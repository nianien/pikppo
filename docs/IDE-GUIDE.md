# ASR Calibration IDE - 操作手册

## 1. 概述

ASR Calibration IDE 是 Dubora 配音流水线的字幕校准工具。ASR 自动识别完成后，通过 IDE 可视化校准说话人、文本和时间轴，校准完成后保存（自动写入 DB）并运行下游流水线，接入翻译和配音流程。

**适用角色**：字幕校准员、运维人员

**所处位置**：

```
Stage:  提取      识别              翻译              配音        合成
Phase:  extract → asr → parse  →  translate  →  tts → mix  →  burn
Gate:                        ↑              ↑
                      source_review   translation_review
                         你在这里
```

## 2. 安装与启动

### 2.1 环境要求

- Python 3.9+
- Node.js 18+（仅首次构建前端需要）
- 浏览器：Chrome / Edge / Firefox

### 2.2 安装

```bash
# 安装 Python 后端依赖
make install-web

# 构建前端（仅首次或前端更新后需要）
cd web
npm install
npm run build
cd ..
```

### 2.3 启动

```bash
# 启动 Web 服务器（默认端口 8765）
vsd-web serve

# 指定端口
vsd-web serve --port 9000

# 另开终端启动 worker（执行 pipeline 任务）
vsd-pipeline worker
```

启动后浏览器访问 **http://localhost:8765**。

## 4. 基本操作流程

### 4.1 选择剧集

1. 点击顶部下拉框，选择剧名和集数
2. 系统自动从 DB 加载 cues 数据
3. 视频和段落列表同时加载

### 4.2 播放与定位

- **点击视频**：播放/暂停
- **点击段落**：跳转到该段开头播放
- **点击时间轴**：跳转到指定时间
- **点击进度条**：跳转到指定位置
- 播放时段落列表自动跟随高亮（绿色左边框 + 背景）

**配音视频切换**：当合成阶段（burn）完成后，视频右上角出现 **Original / Dubbed** 切换按钮，可在原始视频和配音视频间切换对比。

### 4.3 编辑文本

段落列表中每行显示中文原文和英文翻译两行：

- **双击中文文本**：编辑中文原文，修改后 **Enter** 确认，**Esc** 取消
- **双击英文文本**：编辑英文翻译，操作同上
- 编辑后底部状态栏显示 "Unsaved changes"

### 4.4 修改说话人

每个段落行上有**彩色说话人徽章**，直接点击即可修改：

1. 点击说话人徽章 → 弹出下拉菜单
2. 顶部搜索框可按前缀筛选角色
3. 上下箭头导航，Enter 选择
4. 如果输入的文本不匹配任何现有角色，底部出现 **"+ Create ..."** 选项
5. 选择「创建」会立即将新角色添加到 DB roles 表并分配给该段落

**快捷方式**：选中段落后按 **数字键 1-9** 快速切换（对应说话人列表顺序）。

### 4.5 修改情绪

每个段落行上有**彩色情绪徽章**，直接点击即可修改：

1. 点击情绪徽章 → 弹出下拉菜单
2. 列出所有可用情绪（带颜色圆点和支持语言标注）
3. 上下箭头导航，Enter 选择

**快捷方式**：选中段落后按 **N**=neutral, **A**=angry, **S**=sad, **I**=happy, **E**=surprised, **F**=fearful。

### 4.6 调整时间轴

**精细微调**（推荐）：
- 选中段落后，**Alt + 左/右箭头**：起始时间 +-50ms
- **Shift + Alt + 左/右箭头**：起始时间 +-200ms
- **Cmd/Ctrl + Alt + 左/右箭头**：结束时间 +-50ms
- **Shift + Cmd/Ctrl + Alt + 左/右箭头**：结束时间 +-200ms

**时间轴拖拽**：
- 选中段落后，在时间轴上拖拽段落左右边缘（鼠标变为 `col-resize`）
- 最小段落长度 100ms

### 4.7 拆分与合并

- **Cmd/Ctrl+B**（Split）：在当前播放位置拆分选中段落为两段
- **Cmd/Ctrl+M**（Merge）：将选中段落与下一段合并

**操作步骤（拆分）**：
1. 选中目标段落
2. 播放视频，在需要拆分的位置暂停
3. 按 Cmd/Ctrl+B，段落在播放头位置一分为二

### 4.8 插入段落

- **Cmd/Ctrl+I**：在当前播放位置插入一个空白段落
- 也可通过右键菜单选择 "Insert Before" 或 "Insert After"
- 插入后自动进入文本编辑模式

### 4.9 删除段落

- 选中段落后按 **Delete** 或 **Backspace**
- 删除后自动选中相邻段落

### 4.10 段落类型切换

段落有两种类型：**speech**（语音）和 **singing**（歌唱）。

- 歌唱类型的段落显示粉色 `singing` 标签
- 点击该标签或右键菜单可切换类型
- 歌唱段落在 TTS 阶段会使用不同的合成策略

### 4.11 右键菜单

**段落列表右键菜单**：
| 操作 | 快捷键 | 说明 |
|------|--------|------|
| Split at Playhead | Cmd+B | 在播放头位置拆分（播放头不在段落内时禁用） |
| Insert Before | - | 在当前段前插入空白段 |
| Insert After | - | 在当前段后插入空白段 |
| Merge with Next | Cmd+M | 与下一段合并（最后一段时禁用） |
| Set as Singing/Speech | - | 切换段落类型 |
| Delete | Backspace | 删除段落 |

**时间轴右键菜单**（点击段落）：
| 操作 | 快捷键 | 说明 |
|------|--------|------|
| Split Here | Cmd+B | 在右键位置拆分 |
| Merge with Next | Cmd+M | 与下一段合并 |
| Delete Segment | Backspace | 删除段落 |
| Undo | Cmd+Z | 撤销 |
| Redo | Shift+Cmd+Z | 重做 |

**时间轴右键菜单**（空白区域）：
| 操作 | 快捷键 | 说明 |
|------|--------|------|
| Insert Segment Here | Cmd+I | 在此位置插入段落 |
| Undo | Cmd+Z | 撤销 |
| Redo | Shift+Cmd+Z | 重做 |

### 4.12 保存

- 按 **Cmd/Ctrl+S** 或点击顶部 Save 按钮（修改后 2 秒也会自动保存）
- 保存时调用 `diff_and_save()` 写入 DB，cv 自动递增，服务端自动检测重叠
- 数据保存到 SQLite DB (cues 表)

### 4.13 导出

通过 API 调用 `POST /api/episodes/{drama}/{ep}/export`，生成字幕文件：

| 文件 | 路径 | 用途 |
|------|------|------|
| `zh.srt` | `output/` | 中文字幕文件（从 DB cues.text 导出） |
| `en.srt` | `output/` | 英文字幕文件（从 DB cues.text_en 导出） |

## 5. 流水线面板

流水线面板（PipelinePanel）位于视频播放器下方，提供流水线的可视化操作。

### 5.1 阶段状态

将 7 个 Phase 聚合为 5 个可视化阶段：

| 阶段 | 名称 | 包含 Phase |
|------|------|-----------|
| extract | 提取 | extract |
| recognize | 识别 | asr, parse |
| translate | 翻译 | translate |
| dub | 配音 | tts, mix |
| compose | 合成 | burn |

阶段状态用颜色区分：
- **灰色**：pending（未运行）
- **蓝色脉冲**：running（运行中）
- **绿色**：succeeded（成功）
- **红色**：failed（失败）

阶段之间的**门控（Gate）**：
- `source_review`（识别→翻译之间）：ASR 校准审核点
- `translation_review`（翻译→配音之间）：翻译审核点
- 门控状态：pending（灰）→ awaiting（黄色脉冲）→ passed（绿色）

### 5.2 运行操作

右侧按钮根据当前状态显示不同操作：
- **开始**：所有阶段尚未运行
- **重试**：有阶段失败
- **继续**：有门控等待审核
- 运行中显示 **Cancel** 按钮

运行前自动保存未保存的修改。

**右键阶段芯片**：可选择从该阶段重新运行（`从「阶段名」重跑`）。

### 5.3 日志查看

运行时面板底部展开日志区域，实时显示 SSE 推送的流水线日志，自动滚动到最新行。

## 9. 典型工作流程

### 9.1 新集校准

```bash
# 1. 先跑到 parse（含 ASR + 后处理）
vsd-pipeline run 剧名 集号 --to parse

# 2. 启动 IDE
vsd-web serve

# 3. 浏览器打开 http://localhost:8765
#    选择剧集 → 自动从 DB 加载 cues

# 4. 校准工作：
#    - 逐段检查文本，双击修正错别字
#    - 点击说话人徽章修正角色，或用 1-9 快速切换
#    - 点击情绪徽章修正情绪
#    - 拆分/合并错误分段
#    - Cmd/Ctrl+S 保存（或等待自动保存）

# 5. 直接在面板运行下游
#    PipelinePanel → 点击「继续」→ 自动从 translate 跑到 burn
```

### 9.2 批量校准

```bash
# 1. 批量跑到 parse
vsd-pipeline run 剧名 1-20 --to parse

# 2. 启动 IDE，逐集校准
vsd-web serve

# 3. 每集校准完保存后，可在 PipelinePanel 直接运行下游
#    或命令行批量跑：
vsd-pipeline run 剧名 1-20 --from translate --to burn
```

### 9.3 返工修正

如果配音后发现仍需修改：

1. 回到 IDE 修改
2. Cmd/Ctrl+S 保存
3. PipelinePanel 右键对应阶段 → 从该阶段重跑
4. 或命令行：`vsd-pipeline run 剧名 集号 --from translate --to burn`

## 10. Voice Casting（声线分配）

Voice Casting 是独立于 ASR 校准的全屏视图，用于管理 DB roles 表——将角色绑定到 TTS 音色。

### 10.1 进入 / 退出

- 点击主界面右上角 **Voice Casting** 按钮进入
- 进入后主界面完全切换（隐藏剧集选择和 ASR 相关控件）
- 点击左上角 **← Back** 返回 ASR IDE

### 10.2 界面布局

```
+--------------------------------------------------------------+
|  ← Back | Voice Casting | [剧名 v] |           | Save        |
+--------------------------------------------------------------+
|  ▶ Playing: Glen (trial)  [====audio====]  Download           |
+-------------------+------------------------------------------+
| Roles             | Voice Catalogue                           |
|                   | Category: [All] [有声书] ...  Gender: ... |
| ● PingAn          |                                           |
|   Glen            | ○ ▶ Candice  女/青年  happy neutral  [Try] |
| ○ BaiYe           | ● ▶ Glen     男/青年  happy neutral  [Try] |
|   (none)          |    ┌─ inline synthesis ──────────────┐    |
|                   |    │ Emotion [▼] Text [___] [Synth]  │    |
| Default Roles     |    │ ▶ happy "I never thought..."    │    |
| ○ male            |    └─────────────────────────────────┘    |
|   Glen            | ○ ▶ Sylus    男/青年  happy neutral  [Try] |
| ○ female          |                                           |
|   (none)          |                                           |
+-------------------+------------------------------------------+
```

| 区域 | 说明 |
|------|------|
| Header | 独立 header，含剧名下拉框（只选剧、不选集）、Save 按钮 |
| 播放条 | 全局音频播放器，试听 / 合成结果共用 |
| 左栏 | 角色列表（来自 DB roles 表），蓝色高亮选中角色 |
| 右栏 | 音色目录，支持 Category / Gender 筛选 |

### 10.3 操作流程

#### 分配音色

1. 顶部下拉框选择**剧名**
2. 左栏点击角色（如 "PingAn"），进入分配模式
3. 右栏浏览音色，点击 ▶ 试听官方 trial
4. 点击目标音色卡片 → 该音色自动分配给当前角色（蓝色圆点标记）
5. 点击 **Save** 写回 DB roles 表

**自动滚动**：如果被选中的角色已有绑定音色，右栏会自动滚动到该音色卡片并居中显示。

#### 自定义合成试听

1. 点击音色卡片右侧的 **Try** 按钮 → 展开内联合成面板
2. 选择 **Emotion**（从该音色支持的情绪列表中选择）
3. 输入试听文本（有默认文本，可自定义）
4. 点击 **Synthesize** → 合成完成后自动播放
5. 面板下方显示该音色的**合成历史**，点击 ▶ 可回放
6. 再次点击 **Try** 收起面板；展开另一个音色时上一个自动收起

### 10.4 注意事项

- roles 是**剧级别**配置（DB roles 表），所有集共享同一份声线映射
- Voice Casting 不涉及剧集选择，只需选择剧名
- 合成试听会调用 VolcEngine TTS API，需要配置 `DOUBAO_APPID` / `DOUBAO_ACCESS_TOKEN`
- 合成结果缓存在 `.cache/voice-preview/`，相同 voice + text + emotion 不会重复调用 API

## 11. 数据存储说明

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| cues (原子段) | DB cues 表 | IDE 编辑的主要对象，含 text/text_en/speaker/emotion/timing |
| utterances (分组) | DB utterances 表 | 由 `calculate_utterances()` 自动管理，TTS 缓存 |
| roles (角色声线) | DB roles 表 | Voice Casting 页面编辑，剧级共享 |
| dictionary (术语) | DB dictionary 表 | 人名/术语词典 |
| `asr-result.json` | `input/` | ASR 原始输出（只读，不要手动修改） |
| TTS 音频 | `derived/tts/segments/` | 逐句合成的 WAV 文件 |
| `en.srt` | `output/` | burn 阶段从 DB cues.text_en 生成 |

**注意**：
- DB (`data/db/dubora.db`) 是所有元数据的 SSOT
- 编辑 cue 后自动保存到 DB，cv (content version) 自动递增
- 原始 `asr-result.json` 永远不会被修改

## 12. 常见问题

### Q: 打开 IDE 后看不到任何剧集？

检查 `--videos` 参数指向的目录结构是否正确：
```
videos/
  剧名/
    1.mp4
    dub/
      1/
        input/
          asr-result.json
```

### Q: 选择剧集后没有数据？

该集尚未运行 ASR + parse 阶段，先执行：
```bash
vsd-pipeline run 剧名 集号 --to parse
```

### Q: 视频播放不了？

- 确认视频文件位于 `videos/剧名/集号.mp4`（与 dub 目录同级）
- 浏览器需支持 H.264 编码
- 非 faststart 的 MP4 会自动 remux（首次访问较慢）

### Q: 保存后如何确认已保存成功？

查看底部状态栏：
- "Unsaved changes"（黄色）= 有未保存的修改
- "Saved"（绿色）= 已保存到 DB

### Q: 如何运行下游流水线？

两种方式：
1. **IDE 内**：PipelinePanel 点击运行按钮（自动保存后运行）
2. **命令行**：`vsd-pipeline run 剧名 集号 --from translate --to burn`

### Q: 如何回退到 ASR 原始结果？

在 PipelinePanel 中右键 "识别" 阶段，选择从 parse 重跑。parse 会清空旧 cues 并从 `asr-result.json` 重新导入。
