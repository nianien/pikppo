## 一、跨能力设计总则

这些是贯穿所有应用能力的不可妥协约束，新增能力先过一遍这张清单。

1. **对话优先，反馈分流**：完成一切任务的主入口是对话。操作型瞬时反馈（已复制、已转发、已删除等）走 toast，**对话流里只留对话**，不灌系统气泡污染上下文。
2. **归属即红线（local-first）**：
   - 用户自管的结构化数据（日历、记忆、角色、群组、消息、设置）→ **本地** drift / Repository，离线可用、数据不出端。
   - 需要第三方 API Key / OAuth 凭据的能力（汇率、未来的股票/天气/邮件）→ **MCP**，凭据放服务端，绝不下发客户端。
   - 需要知识广度、本地办不到的查询（解释、翻译）→ **LLM 直调**。
3. **私聊 = 群聊的特例**：私聊、群聊每个角色都走统一的 `_generateReply`，由**模型自行决定**要不要 emit tool_use。不为某种聊天单开"锁死纯 chat"的岔路——角色能力随角色走，不随会话类型变。
4. **外部 LLM 内容查询的两条红线**（解释 / 翻译等把聊天内容发给 LLM 的便捷功能）：
   - **必须用户显式触发**（点菜单），绝不后台自动扫描内容。
   - **只带最小必要上下文**（那个词 + 它所在的一两句），不把整段对话甩出去。
   - 结果用**弹窗**呈现，不进对话历史。价值在省去"复制→切到 AI→粘贴→问"的摩擦，而非新增外泄通道。
5. **失败可重试且不污染历史**：失败回复以 error 气泡呈现（不落库、不进 LLM 历史），下方"重试"重跑该轮（群聊只重跑失败的那个角色）。

## 二、能力清单

### 2.1 私聊对话

- **场景**：选一个角色直接对话，助理基于角色 Prompt + 个人记忆 + 日程上下文回复。
- **行为**：消息按会话懒加载 + 分页；模型有 tool 能力且有可用工具时进 agent loop，由模型决定调本地/MCP 工具；否则退化纯 chat。
- **约束**：消息历史只喂 `kind=='chat'` 的；error / 工具状态 / 提醒是 UI-only，不进历史。
- **归属**：本地（消息存 drift）+ LLM + 按需 工具。
- **状态**：✅

### 2.2 群聊

- **场景**：多角色协作，一条消息可由多个相关角色依次回复。
- **行为**：`@提及` 命中则只让被 @ 的角色回；否则 LLM 路由判断哪些角色相关。被选中的角色逐个走与私聊**同一套** `_generateReply`（含工具能力），回复之间有节奏停顿避免刷屏。
- **约束**：路由是一次性轻量判断；每个角色回复独立 try/catch，失败只挂该角色的可重试 error 气泡，其余成功回复不动。
- **归属**：本地 + LLM + 按需 工具。
- **状态**：✅

### 2.3 消息操作

- **场景**：对单条 / 多条消息做复制、转发、分享、删除。
- **行为**：
  - **复制**：系统选择能力，复制选中文本。
  - **转发**：转发整条消息到任一角色私聊 / 群聊。
  - **分享**：调系统分享面板（share_plus）分享消息文本。
  - **多选删除**：长按消息 → 菜单「多选」进入选择模式；顶栏「已选 N / 删除」，二次确认。删的是**单/多条消息**，不是整个会话；私聊群聊一致。
  - **重试**：见总则 5。
- **约束**：长按文字弹**横排多行**的微信式深色菜单（自绘，避免系统溢出竖排）；图片/文件长按走底部面板。
- **归属**：本地（删除写 drift + 附件清理）/ 系统（分享、复制）。
- **状态**：✅

### 2.4 选词能力：解释 / 翻译

- **场景**：聊天里遇到不懂的专有名词或外语，想就地弄懂，不必切到别处再问。
- **行为**：
  - **解释**：选中词 → 菜单「解释」→ 带"词 + 所在一两句语境"一次性调模型 → 弹窗返回解释（带复制）。
  - **翻译**：整条 / 选中文本 → 一次性调模型译成用户首选语言（本就是该语言则译成英文）→ 弹窗返回。
- **约束**：严格遵守总则 4（显式触发、最小上下文、弹窗不入历史）。私聊群聊都能用。这不是字典（查表解决不了专有名词 + 语境），是用户显式触发的上下文化 LLM 查询。
- **归属**：LLM 直调。
- **状态**：✅

### 2.5 日历 / 日程

- **场景**：「明天下午两点开会，加到日历，提前 15 分钟提醒」；或在应用面板手动管理日程。
- **行为**：
  - 应用面板日历页：日期选择 + 当日事件列表，事件 sheet 增改删。
  - 对话自动提取：用户在聊天里提到日程 → 模型 emit `calendar_create_event` 等工具 → 创建事件。
  - **UI 与 LLM 工具共享同一条本地写入路径**（`CalendarRepository`），含盖戳 / 软删 / reminder 调度。
- **约束**：日历是**本地化**能力，不走 MCP（pikppo-mcp 即便仍留 calendar 工具，客户端也不调）。
- **归属**：本地（drift `CalendarEventRows` + `CalendarRepository`）。
- **状态**：✅

### 2.6 汇率

- **场景**：查实时汇率、做换算、看一段时间走势。
- **行为**：
  - 换算：输入金额 + 选币种，显示换算结果 + 正向/反向汇率 + 更新时间（本地时区中文相对时间）。
  - 走势：7 / 30 / 90 天折线，带区间起点基准虚线（一眼看涨跌）、涨跌幅、低/高。
  - 常用对：用户可**增删**收藏对（持久化），当前对高亮一键切换。
  - 下拉刷新。
- **约束**：汇率是**当前唯一真正走 MCP 的功能**（需第三方 API Key，凭据放服务端）。
- **归属**：MCP（`convert_currency` / `get_exchange_rate` / `get_exchange_trend` / `list_exchange_rates`）；收藏对存 SharedPreferences。
- **状态**：✅

### 2.7 工具调用反馈

- **场景**：模型调工具时让用户知道"在做什么"，但不留痕迹污染对话。
- **行为**：agent loop 执行工具期间显示**瞬时提示**「正在调用工具：<中文名>」（接口名→中文映射），工具跑完即清空。
- **约束**：瞬时态，不落库、不进 LLM 历史。私聊群聊一致。
- **归属**：本地 UI 状态。
- **状态**：✅

### 2.8 对话内富图表卡片

- **场景**：在聊天里问"美元对人民币最近一个月走势"，比起一段文字，直接出一张走势图更直观。
- **行为**：agent loop 执行到**可视化工具**（首发 `get_exchange_trend`）时，把图表数据**挂到模型这一轮的文字回复上**——**同一条消息、同一个气泡、同一个头像**（图在上、文字在下）。**图 + 文都留**（图直观、文字仍是可选中/复制/转发/翻译的正常文本）。卡片**先缓存**，等文字产出时随 `onFinal` 一起落，不另起气泡、不割裂。出图的工具结果会附一句系统提示，**引导模型只给一两句话结论**（趋势/涨跌幅/区间/最新值），不复述每日明细或画 ASCII 表。
- **可扩展框架**：`工具名 → 卡片类型` 注册表（`lib/utils/chart_cards.dart`）+ 渲染端按 type 选 widget（`message_bubble.dart` 的 `_chartBodyFor`）。新增股票/天气出图，加一行映射 + 一个 widget 分支即可。绘制组件 `ExchangeTrendChart` 与汇率页共用。
- **数据模型**：图表存消息的独立列 `chart_data`（drift v6，JSON 数组 `[{type,data}]`）；`content` 保持纯文本——进 LLM 历史、可复制/转发，**图表不进历史、不入会话列表预览**。数据不足以成图时不挂图，只留文字。整条消息**参与多选删除**（一条消息一个 id）。
- **归属**：本地渲染（数据来自对应工具：汇率走势走 MCP）。
- **状态**：✅（汇率走势）

### 2.9 提醒呈现

- **场景**：到点提醒用户某个日程。
- **行为**：提醒以**聊天消息**形态出现在对应角色私聊里 + **系统通知**双轨触达；一条事件归属单一角色。
- **约束**：本文只覆盖"呈现形态"；**单角色路由 / OS 通知预调度的机制**属 agent 路由，见 `技术方案设计.md` 与 `data-architecture.md`。
- **归属**：本地。
- **状态**：⏳（系统通知双轨进行中）
